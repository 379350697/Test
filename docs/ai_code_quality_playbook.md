# AI 代码质量审查与迭代模板（QuantX）

## 1) 当前代码库审查结论（快照）
- **单元测试**：存在 `tests/test_quantx.py`，覆盖了回测、优化、执行、监控、AB 测试和策略动态加载等主流程。
- **覆盖率要求**：已在 `pyproject.toml` 配置 `pytest-cov` 与最低门槛（`--cov-fail-under=60`），后续建议持续提升阈值。
- **边界情况处理**：策略层对样本不足、价格非正、ATR/ADX 不可计算等已做基础防御。
- **性能/安全约束**：存在并行回测与风控参数；已补充安全扫描门禁（`pip-audit` + `bandit`），已新增性能冒烟测试与 CI 门禁，并覆盖 backtest + live/rebalance/risk 关键路径。
- **错误处理与日志**：有部分模块提供状态与日志分析能力，但缺少统一日志规范与分级错误码约束。

## 2) 强制质量门禁（建议落地到 CI）
1. `pytest` 必须全绿（已接入 `pytest-cov`）。
2. `ruff check .` 必须全绿。
3. `mypy quantx tests` 关键模块零错误（可先分阶段豁免第三方依赖）。
4. 覆盖率：当前门槛 `line >= 60%`（已配置），目标门槛 `line >= 80%`；核心路径（`backtest/strategies/execution`）目标 `>= 90%`。
5. 安全门禁：`python -m pip_audit`（依赖漏洞）与 `python -m bandit -q -r quantx`（静态安全扫描）必须执行并全绿。
6. 新功能必须附带：
   - 至少 1 个正常路径测试
   - 至少 2 个边界/异常测试
   - 风险评估（性能/资金安全/回撤约束）

## 3) 分步执行计划（禁止“一口气生成”）
- **Step A：需求澄清**
  - 明确交易场景、时间粒度、资金规模、可接受回撤、吞吐与时延目标。
- **Step B：方案评审**
  - 先输出接口与数据流，不写实现。
  - 列出失败场景和回滚策略。
- **Step C：最小实现**
  - 只交付最小可运行增量。
- **Step D：测试补齐**
  - 加边界测试、回归测试、性能冒烟测试。
- **Step E：静态检查**
  - lint + type check + 复杂度检查。
- **Step F：审查与迭代**
  - 输出审查意见，按高/中/低优先级循环优化。

## 4) 给 AI 的模板化提示词（可复用）

### 模板 1：需求澄清
```text
你是 QuantX 项目的代码助手。先不要写代码。
请先完成：
1) 需求重述（含目标/非目标）
2) 输入输出契约
3) 风险与边界场景列表（至少 10 条）
4) 验收标准（功能、性能、安全、可观测性）
输出用 Markdown 表格。
```

### 模板 2：分步实现
```text
按以下顺序执行，不可跳步：
Step 1. 先给变更计划与影响文件清单
Step 2. 仅实现最小闭环代码
Step 3. 补齐单元测试（正常+边界+异常）
Step 4. 运行 pytest/ruff/mypy/pip-audit/bandit 并贴结果摘要
Step 5. 运行性能冒烟测试并记录耗时阈值
Step 6. 自查并列出剩余风险与下一轮优化项
```

### 模板 3：审查与循环优化
```text
请作为 Reviewer 审查本次改动：
- 正确性
- 边界处理
- 性能复杂度
- 安全/风控
- 可维护性（命名、耦合、重复）
输出：
A. 必改问题（阻断合并）
B. 建议优化（可后续迭代）
C. 最小修复补丁建议
随后进入下一轮修复，直到 A 类问题为 0。
```


## 5) 下一步执行模板（需求上下文 + 强制质量标准 + 循环优化）

### 5.1 需求上下文输入模板（必须先填）
```text
【业务目标】
【非目标】
【输入数据来源/格式/规模】
【输出定义与验收指标】
【运行环境与资源约束（CPU/内存/时延）】
【风险约束（最大回撤、单笔风险、杠杆上限）】
【安全约束（密钥、权限、审计）】
【回滚策略】
```

### 5.2 强制质量标准（执行前确认）
- 功能：需求用例 100% 可追踪到测试。
- 质量门禁：`pytest` / `ruff` / `mypy` / `pip-audit` / `bandit` 必须执行并附摘要。
- 测试结构：每个功能至少 `1` 正常 + `2` 边界/异常。
- 变更说明：必须包含风险评估与回滚方案。

### 5.3 AI 分步协作模板（禁止一口气出最终实现）
```text
阶段 1：仅输出设计与风险清单，不写代码。
阶段 2：输出最小可行补丁（MVP）与对应测试。
阶段 3：运行并粘贴 pytest/ruff/mypy/pip-audit/bandit 结果。
阶段 4：作为 reviewer 列出阻断项与建议项。
阶段 5：按阻断项修复并重复阶段 3-4，直到阻断项为 0。
```

### 5.4 代码审查输出模板（每轮固定格式）
```text
[Review Round N]
A. 阻断问题（必须修）
B. 重要建议（建议本轮修）
C. 可延期优化
D. 测试结果摘要（pytest/ruff/mypy/pip-audit/bandit）
E. 风险结论（可发布/不可发布 + 原因）
```
